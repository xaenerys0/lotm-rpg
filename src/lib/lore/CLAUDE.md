# Lore Database

Structured lore data for RAG retrieval by the AI integration layer. Each entry is a semantic chunk (300-800 tokens) representing one logical entity.

> **Canon is verified against `corpus/`, never memory.** Before authoring or editing any lore entry, fact-check names, pathways, affiliations, relationships, and events against the LOTM wiki dump (`corpus/wiki/lordofthemystery_pages_current.xml`) and novel (`corpus/novel/…EPUB`). The corpus is Git LFS — if a file is a ~130-byte pointer, run `git lfs install && git lfs pull` first (`apt-get install -y git-lfs` if needed). The corpus outranks an issue's parenthetical hint. See the repo-root `CLAUDE.md` "Canon & Source Material" section.

## Structure

- `types.ts` — `LoreEntry` and `LoreCategory` type definitions.
- `retrieval.ts` — Runtime retrieval (issue #63): `retrieveChunks(query, options)`
  embeds the query with the save's **locked** embedding model (per-character
  model lock — mismatches throw) and calls the gated `match_source_chunks` RPC.
  The **timeline gate** (`canon_order <= canonPosition`) and **concealment
  gate** are enforced server-side and re-checked client-side as defense in
  depth. `createSupabaseChunkMatcher(client)` wraps a Supabase client into the
  injectable RPC seam; `retrievalChunkIds(chunks)` extracts the ids recorded on
  the turn record (determinism/debuggability); `toPgVector` serializes query
  vectors. The **epoch gate** (`RetrieveChunksOptions.epoch`, character epoch
  isolation) rides into the RPC as `p_epoch` and is re-checked client-side: a
  non-Fifth character never retrieves Fifth-Epoch canon chunks (`null` = no
  limit, eval/operator only). This **extends** the curated `lore_entries`
  injection — curated guardrails are injected first (issue #64); retrieved chunks
  fill the rest. **`concealmentTierForSequence(seq)`** maps the character's
  current sequence to the retrieval concealment ceiling (progressive spoiler
  disclosure: Seq 9-8 → 0, 7-5 → 1, 4-2 → 2, 1 → 3, 0 → 4), passed as
  `maxConcealmentTier`. **Retrieval is now wired into play**: the game loop calls
  it best-effort each turn (`components/game/lore-retrieval-client.ts` →
  `buildRetrievalQuery` from `@/lib/game`), feeding `generate({ retrievedChunks })`;
  it no-ops to `[]` when no embedding endpoint/corpus is available, so curated
  lore always stands alone as the floor.
- `selection.ts` — `selectCuratedLore(pathway, location, budget, epoch?, sequenceLevel?)`: greedy first-fit over pathway → epoch-setting → city lore, deduped, epoch-gated, and (when `sequenceLevel` is given) **sequence-gated** via `passesSequenceGate` — a pathway rung's lore is revealed only once the character has reached its earliest (highest-numbered) sequence, mirroring the glossary's `revealAtSequence` progressive disclosure. Entries with no `sequences` (geography/era/orgs) always pass.
- `epochs.ts` — Epoch starting points (issues #26/#29): `EPOCHS` (all five — player-safe summary, narrator `toneDirective`, `startingLocation`, `openingBeat`, `dangerModifier`; the Fifth is the baseline for **tone** — empty `toneDirective` — but, like every other epoch, now carries an **awakening `openingBeat`** that continues from the prologue's final potion so the chronicle never opens by abruptly announcing the character's pathway), `getEpoch` (unknown → Fifth), `isFifthEpoch(epochId)` (the single shared "is this the default/Fifth era" rule — the only era with the per-city atlas/`currentCity`/travel model; reused by `@/lib/game` `place-graph.ts` and `location.ts` so they can't disagree), `epochNarrationDirective`, `epochOpeningBeat`. Threaded into prompt assembly as the `## Epoch` system layer (PromptInput.epochContext) and into the first scene's player action. **`passesEpochGate(entryEpoch, characterEpoch)`** — the epoch content gate (character epoch isolation): an **untagged** entry (`epoch` undefined/null) is **universal** and passes for everyone; a **tagged** entry passes only on **exact** epoch equality; an absent character epoch defaults to the Fifth. Mirrors `passesTimelineGate`'s "timeless rows always pass" shape and is the single rule reused by `selection.ts`, `glossary.ts`, `retrieval.ts`, and the SQL `match_source_chunks` RPC — so a non-Fifth character never sees Fifth-Epoch lore, chunks, glossary terms, or map regions (and vice versa).
- `epoch-first.ts` / `epoch-second.ts` / `epoch-third.ts` / `epoch-fourth.ts` — Rich, canon-grounded curated lore for the four pre-Iron-Age epochs (`FIRST_EPOCH_LORE` … `FOURTH_EPOCH_LORE`; grounded in the LOTM wiki/novel — Age of Chaos, Dark Epoch, Cataclysm Epoch, Epoch of the Gods). Each set's `metaphysics`-category entries are the era's **setting overview** (injected by `selectCuratedLore` via `getLoreByEpochSetting`); `location`-category entries surface in the gazetteer/discovery. Every entry is tagged with its `epoch`.
- `gazetteer.ts` — Epoch- AND city-keyed map atlas (character epoch isolation, issue #101): `gazetteerForEpoch(epoch, cityId?, accessFlags?)` returns, for the Fifth Epoch, the **current city's** districts (Tingen/Backlund/Trier/Bayam plus the start regions Pritz/Enmat/Feysac and the wider-Loen city Constant — each with its own public district list) with a header naming that city and "farther afield" travel to the OTHER Fifth cities; an unrecognised/absent `cityId` falls back to Tingen. **Continent filter (world build-out, issue #130):** `GazetteerFartherCity.continent?: "central"|"forsaken-land"` (absent = central) plus the two content-less Forsaken roster cities (`silver-city`/`giant-kings-court`, no districts — they fall back to `[]`, not Tingen) let the farther-afield list be filtered to same-continent cities, plus cross-continent ones only when the character holds the `CONTINENT_CROSSING_FLAG` (`accessFlags`, threaded from `mapAtlasFor`) — so a central character never sees the Forsaken cities in their travel panel, and vice-versa. **Crossing chokepoint + per-city flags (issues #132/#133):** `continentReachable` now gates each FORSAKEN city by its OWN `GazetteerFartherCity.requiresFlag` — Giant King's Court (`dream-world-passage`, the shared gate), the City of Silver (`silver-city-passage`), and Moon City (`moon-city-passage`) — so a Silver native sees the Court but never Moon City, a Moon native sees the Court but never Silver, and an outsider at the Court sees neither native city until admitted. A CENTRAL city shows to a mainlander, and from the Forsaken side only at the crossing city `GAZETTEER_CROSSING_CITY = "giant-kings-court"` holding the passage (leaving routes through the Court). The per-city `requiresFlag` literals + the crossing-city literal are lore-local, reconciled by a test against the game-layer `City.accessGate.requiresFlag`/`CROSSING_CITY` (the gazetteer must not import the game/AI layers), mirroring the `PROLOGUE_AFFINITY_REGIONS` pattern. Moon City carries its own district list (the fog-watch walls, the priests' hold, the lower warrens). Earlier epochs return their static, public region list (travel disabled — the CITIES model is Fifth-Epoch only). `uncertainFifthGazetteer()` (Backlund location sync) is the neutral atlas the map shows when a Fifth-Epoch city genuinely can't be resolved — no districts (so nothing is mis-pinned) but every **central** city listed as travel-reachable (the Forsaken cities stay hidden — issue #130), so setting out re-anchors `currentCity` and self-heals; the game-layer resolver `@/lib/game/location.ts` (`mapAtlasFor`) chooses it instead of defaulting to Tingen. The `cityId` is supplied by the caller (`cityIdFromLocation(location)` in `map-panel.tsx`) — the gazetteer must not import the game-layer city resolver. Player-safe public knowledge only; the narrator-only curated lore must not bleed in. Consumed by `map-panel.tsx`. (Movement reachability — `@/lib/game/place-graph.ts`, issue #101 — keys on `cityIdFromLocation`, NOT on these district keywords, so it never false-attributes one city's sub-location to another.)
- `glossary.ts` — In-game glossary (issue #14): `GLOSSARY_TERMS` with **progressive disclosure** via `revealAtSequence` (9-8 basics at the start; rituals/organisations ~8-7; demigod concepts at 5; Sefirah-tier mysteries at 4) **and an epoch gate** (character epoch isolation) via an optional `epoch` tag. `glossaryForSequence(level, epoch?, accessFlags?)` and `sealedTermCount(level, epoch?, accessFlags?)` apply epoch, sequence **and capability gates**; the sealed count is computed over the **epoch- and capability-applicable** universe only, so an other-epoch or flag-gated term's existence is never leaked. Fifth-specific terms (Loen, Tingen, the churches, the Saint honorific, plus the Constant/Midseashire and Intis Republic/Eternal Blazing Sun/Aurora Order terms from world build-outs 5-6) are tagged `epoch: 5`; universal mechanics (Beyonder, Sequence, Pathway, …) stay untagged; each non-Fifth epoch has its own era terms (Ancient Gods, Ancient Sun God, Solomon Empire, …). **Capability gate (issues #132/#133):** a term may carry `requiresFlag?: AccessFlag`, shown only to a character who holds that flag — so a mainland Fifth character never sees the sealed continent's vocabulary and the sealed count never hints at it. The Forsaken terms are gated PER CITY: the City of Silver / Silver Knights terms require `"silver-city-passage"`, Moon City / the Gray Fog require `"moon-city-passage"`, and the shared continent terms (Giant King's Court, Sea of Ruins, the dream-world passage) require `"dream-world-passage"` — so a Silver native never sees Moon City's vocabulary and vice versa (mutual unawareness). `getGlossaryTerm(slug)`. Entries are player-safe world-building — no narrator-only lore.

- `start-scenarios.ts` — Varied story openings: `START_SCENARIOS` (per-epoch pool of `StartScenario` = `{id, epoch, location, blurb, openingBeat, pathwayAffinity?, origin?}`) plus the selection seam. `selectStartScenario(epoch, random?)` is a pure, deterministic-under-injected-`random` **uniform** draw (pathway does NOT bias the random draw); `selectStartScenarioForLocation(epoch, location, random?)` honours a player's **preferred** location (the scene still varies among that place's openings; falls back to a random epoch start for an unknown location). Consumed by `character-creation.tsx` → `play-dashboard.tsx` → `createDefaultGameState(..., startScenario)`.

## Conventions

- One entry per logical concept (one NPC, one location, one sequence).
- Target 300-800 tokens per entry for retrieval precision.
- Slugs are URL-safe unique identifiers: `category-qualifier` pattern.
- TypeScript source is the canonical data; SQL seed is generated from it.
- When adding new lore, add to the appropriate `.ts` file and regenerate the seed migration.
