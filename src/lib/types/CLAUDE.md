# Type Definitions

- `rules.ts` — Game rules types (pathways, sequences, characteristics, validation results). Source of truth for the rules engine API surface. `Item.category` has five kinds: the three advancement-ladder reagents (`main-ingredient`, `supplementary-ingredient`, `potion-formula`), `mundane` for ordinary loot (issue #90), and `uniqueness` for the singular apotheosis artifact. The AI item-discovery path may only mint `mundane` (plus the `uniqueness` endgame exception); the reagent kinds are rules-engine-only. Every exhaustive `Record<Item["category"], …>` (marketplace `PRICE_GUIDANCE`, character-sheet labels/glyphs) must cover all five.
- `combat.ts` — Combat system types (issue #10): `Enemy` (with optional `knownAbilities`/`description` surfaced by the intel-gated dossier), `CombatPreparationInput`, `PreparationQuality`, `PathwayMatchup`, `DecisionPoint`/`DecisionOption` (with optional `abilityName`/`artifactItem` for dynamic mid-fight options invoked in the moment), `CombatOutcome`, `Injury`, `CombatResult`, and the serializable `CombatEncounter` state (with an optional `huntTarget` — the potion-preparation hunt objective from issue #84 — plus optional `availableAbilities`/`availableArtifacts` captured at encounter start so a serialized fight regenerates identical dynamic options). Imports `Item`/`BeyonderCharacteristic` from `rules.ts`; `GameState.injuries` (in `@/lib/ai`) references `Injury` from here. The deterministic engine (incl. `enemyIntel`) lives in `@/lib/game/combat.ts`. Hunt-quest state (`HuntState` on `GameSession.hunts`) lives in `@/lib/game/hunt.ts`.
- `database.ts` — Supabase schema types (`Database` interface). Defines `profiles`, `lore_entries`, `source_chunks`, and `chunk_embeddings` tables; the `match_source_chunks` RPC under `Functions`; plus `LoreCategoryEnum` and `SourceChunkSourceEnum`. pgvector columns serialize as `string` (see `chunk_embeddings.embedding`). `source_chunks.epoch` (nullable smallint, character epoch isolation) and the matching `match_source_chunks` `p_epoch` arg + `epoch` return column gate the corpus by epoch (migration `20260614000000`). Cross-device sync tables: `game_sessions` (the full save in a jsonb `data` column + `is_active` pointer), `world_memory` (legacies/echoes jsonb), `user_preferences` (preferences jsonb + `dismissed_hints`), and the `set_active_session` RPC (migrations `20260615000000`–`20260615000200`).

## Conventions

- Keep types in sync with the database schema. When adding migrations, update `database.ts` to match.
- Rules types are pure data — no methods or side effects.
- Use `LawType` union for tagging validation violations by which cosmic law was broken.
