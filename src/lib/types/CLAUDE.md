# Type Definitions

- `rules.ts` — Game rules types (pathways, sequences, characteristics, validation results). Source of truth for the rules engine API surface. `Item.category` has six kinds: the three advancement-ladder reagents (`main-ingredient`, `supplementary-ingredient`, `potion-formula`), `mundane` for ordinary loot (issue #90), `uniqueness` for the singular apotheosis artifact, and `sealed-artifact` for the church-catalogued dangerous relics (the corpus-verified catalogue + helpers live in `@/lib/lore/sealed-artifacts`). The AI item-discovery path may only mint `mundane` (plus the `uniqueness` endgame exception); the reagent kinds are rules-engine-only, and Sealed Artifacts are engine/church-gated — granted via `grantSealedArtifact`/combat spoils, never AI-minted (`partitionDiscoveredItems` blocks them like reagents) and never market-traded/fenced. Every exhaustive `Record<Item["category"], …>` (marketplace `PRICE_GUIDANCE`, character-sheet labels/glyphs) must cover all six. `Item.consumable?` (issue #187 §4.6) is an optional per-item override of the category default resolved by `isConsumable` (`@/lib/game/inventory`) — set only when an item's destroy-on-use behaviour differs from its category (a non-consumable Sealed Artifact, a single-use relic).
- `combat.ts` — Combat system types (issue #10): `Enemy` (with optional `knownAbilities`/`description` surfaced by the intel-gated dossier, and a `bestiaryId` when drawn from the curated catalogue), `CombatPreparationInput`, `PreparationQuality`, `PathwayMatchup`, `DecisionPoint`/`DecisionOption` (with optional `abilityName`/`artifactItem` for dynamic mid-fight options, plus `resolutionLine`/`effectTag` from issue #187), `CombatOutcome`, `Injury`, `CombatResult`, and the serializable `CombatEncounter` state (optional `huntTarget` from #84, plus optional `availableAbilities`/`availableArtifacts`/`availableKit`/`context` captured at encounter start so a serialized fight regenerates identical options). **Combat overhaul (issue #187):** `EncounterFraming` (why/who you fight — `hostile-beyonder`/`mundane-threat`/`mind-controlled`/`lost-control`/`rival-motive`/`coerced`/`beast`) + `EncounterContext` (framing/isKnownPerson/controllerName?/motive?/reconcilable, on the encounter); the expanded `CombatOutcome` (adds `subdued`/`freed`/`talked-down`/`captured`/`spared`); `ResolutionLine` (the non-lethal lines tagged on a `DecisionOption`); and `CombatAbility`/`AbilityKind` (the kit tool — `combatKitFor` in `@/lib/game/combat-abilities.ts`). **Phase 3:** `ControlTier` (`steady`→`spiral`) + transient `controlStrain`/`controlTier`/`highRisk` on `CombatEncounter` and `strainDelta` on `DecisionOption` (the loss-of-control meter); `EnemyStance` (`pressing`/`guarded`/`reeling`/`desperate`) + `DecisionPoint.enemyStance` (reactive stances); and `CombatControlVerdict` + `CombatResult.lossOfControl` (the spiral's setback/permadeath verdict the React layer routes through `concludeChronicle`/`applySetback`). Imports `Item`/`BeyonderCharacteristic` from `rules.ts`; `GameState.injuries` (in `@/lib/ai`) references `Injury` from here. The deterministic engine (incl. `enemyIntel`, `selectOpponents`) lives in `@/lib/game/combat.ts`; the curated bestiary in `@/lib/lore/bestiary.ts`.
- `database.ts` — Supabase schema types (`Database` interface). Defines `profiles`, `lore_entries`, `source_chunks`, and `chunk_embeddings` tables; the `match_source_chunks` RPC under `Functions`; plus `LoreCategoryEnum` and `SourceChunkSourceEnum`. pgvector columns serialize as `string` (see `chunk_embeddings.embedding`). `source_chunks.epoch` (nullable smallint, character epoch isolation) and the matching `match_source_chunks` `p_epoch` arg + `epoch` return column gate the corpus by epoch (migration `20260614000000`). Cross-device sync tables: `game_sessions` (the full save in a jsonb `data` column + `is_active` pointer), `world_memory` (legacies/echoes jsonb), `user_preferences` (preferences jsonb + `dismissed_hints`), and the `set_active_session` RPC (migrations `20260618102315`–`20260618102336`). The `rate_limits` table + `check_rate_limit` RPC (returns `{allowed, remaining, reset_at}[]`) rate-limit the operator-funded embed proxy per user (migration `20260619103613`).

## Conventions

- Keep types in sync with the database schema. When adding migrations, update `database.ts` to match.
- Rules types are pure data — no methods or side effects.
- Use `LawType` union for tagging validation violations by which cosmic law was broken.
